Level 1: AWS Infrastructure Scan

13 boto3 checks across IAM, S3, EC2, and CloudTrail. Use the demo flow to create test misconfigs and scan for ~8 findings.

Demo Environment

Create intentional AWS misconfigs, scan, and view findings - all from this dashboard.

Scanner: checking... Admin: checking...

Requires .env (scanner) and .env.admin (aivar-admin). See README Section 7.

Level 2: Multi-Domain Scan

AWS infrastructure plus API endpoints, dependency CVEs, and secrets scanning. Findings are deduplicated and ranked by business impact.

  • 6 API checks (headers, CORS, rate limit, auth bypass, errors, HTTP methods)
  • OSV dependency CVE scan on test_vulnerable_requirements.txt
  • Secrets scan on test_secrets.env

Level 3: Autonomous Continuous Scanning

Scheduled multi-domain scans with persistent findings, lifecycle tracking, SLA alerts, Slack escalation, and posture trends.

  • Configurable scan schedule (default: every 6 hours)
  • Finding lifecycle: opened, updated, resolved, re-opened
  • Cross-run deduplication and posture score trends
  • Critical escalation via Slack webhook

Requires SLACK_WEBHOOK_URL in .env for escalation. DB: storage/findings.db
